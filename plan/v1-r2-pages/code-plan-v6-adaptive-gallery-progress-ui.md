# v6 代码实现计划（不写代码）：展示尺寸自适配 + 上传进度条可视化（Idea-009 / Idea-010）

-   适用版本：`plan/v1-r2-pages`
-   文档日期：2026-03-28
-   来源想法：`plan/0-idea-seed-backlog.md`（Idea-009 / Idea-010）
-   前置文档：`code-plan-v4-batch-upload.md`、`code-plan-v5-hash-instant-upload.md`
-   目标：在不改变匿名上传和风控主策略的前提下，提升“展示页自适配观感”和“上传过程可视反馈”。

---

## 1. 本次范围

1. 展示页按图片宽高比自动适配 UI（卡片样式、详情布局）。
2. 上传页用进度条替代当前调试文本反馈。
3. 批量上传支持多进度条，并在固定高度滚动窗内展示。
4. 上传页验证码区域视觉收敛：
    - 人机验证转圈区域下方不再显示调试红框；
    - 不再显示“人机认证成功”这类技术提示文案。

---

## 2. 非目标

-   不改动鉴权模型（仍无登录）。
-   不新增 AI 智能裁剪与近似图像识别。
-   不做上传剩余时间预测与网速估算。
-   不在 v6 引入复杂动效编辑器。

---

## 3. 设计原则

1. **优先可读性**：用户只看“进度与结果”，不看原始调试 JSON。
2. **状态一致性**：前端状态机与后端逐项结果严格对齐。
3. **移动端优先**：多进度条必须放入滚动窗，控制纵向占用。
4. **渐进增强**：不破坏 v4/v5 已有接口契约。

---

## 4. 数据与接口适配

## 4.1 展示自适配所需字段

建议列表/详情接口至少可用：

-   `width`
-   `height`
-   `aspect_ratio`（可前端计算，也可后端直接返回）

若历史数据缺失宽高，策略：

-   列表页回退 `unknown` 分类样式；
-   详情页按默认容器展示；
-   后续通过离线回填补齐元数据。

## 4.2 上传进度所需结果字段

沿用 v4/v5：

-   `clientFileId`
-   `ok`
-   `uploadMode`
-   `dedupHit`
-   `errorCode`
-   `message`

用于每条进度条最终状态收敛。

---

## 5. 展示页 UI 自适配规划（Idea-009）

## 5.1 比例分档规则（MVP）

建议阈值：

-   竖图：`aspect_ratio <= 0.8`
-   方图：`0.8 < aspect_ratio < 1.2`
-   横图：`aspect_ratio >= 1.2`
-   未知：宽高缺失

## 5.2 列表页适配

-   竖图：卡片高比更高，避免过度裁切。
-   方图：标准正方视觉权重。
-   横图：降低容器高度，突出横向画面。
-   未知：统一回退样式，避免布局抖动。

## 5.3 详情页适配

-   竖图：图像区域优先，操作区下移。
-   横图：操作区可与图像同屏并排（桌面）/下置（移动）。
-   超长图：限制单屏最大高度，保留滚动查看。

## 5.4 验收要点

-   同批含横/竖/方图时，无明显跳闪与错位。
-   移动端首屏不出现按钮被挤出视区。

## 6. 上传页进度可视化规划（Idea-010）

## 6.1 状态机映射

每文件状态：

-   `hash-checking`
-   `instant-ready`
-   `preparing`
-   `uploading`
-   `finalizing`
-   `success`
-   `failed`
-   `canceled`

## 6.2 进度条展示策略

-   单图：显示 1 条总进度条（阶段切换+百分比）。
-   批量：每个文件 1 条进度条。
-   多条进度条外层容器：
    -   固定高度（如 220~280px）；
    -   `overflow-y: auto`；
    -   保持标题区与操作按钮区固定可见。

## 6.3 调试文案替换策略

-   默认不再展示原始 JSON 文本/红色 debug 面板。
-   失败时仅展示“可读错误摘要 + 可展开详情（可选）”。
-   成功时显示简明结果统计：成功数、失败数、秒传命中数。

## 6.4 人机验证区域收敛（新增约束）

上传界面中 Turnstile 区域要求：

1. 转圈（加载/挑战）区域下方不再显示调试红色框。
2. 不再显示“人机认证成功”提示文案。
3. 验证态仅通过轻量视觉状态体现：
    - 默认：待验证
    - 失败：简短错误提示
    - 成功：不额外占位提示（静默通过）

## 6.5 上传表单文案与控件一致性（2026-03-28 补充）

1. 上传者昵称输入框 placeholder 统一为：`输入你的昵称（不填默认为093）`。
2. 删除上传按钮下方“提示：若昵称留空，会自动使用默认昵称。”文案，避免重复提示。
3. Turnstile 区域下方不再保留常驻说明文案，仅在失败/过期时显示简短错误提示。
4. 美化 `input[type=file]` 的文件选择按钮（圆角、粉色渐变、悬停态），与站点主题统一。

> 说明：若 Turnstile 使用 Cloudflare 官方测试 Site Key，组件可能显示测试用途提示；上线环境应使用正式 Site Key。

## 8.1 视觉回归

-   横图/竖图/方图/超长图各 2 组样例截图。
-   桌面与移动端各一轮回归。

## 8.2 交互回归

-   单图上传完整链路：预检 -> 上传/秒传 -> 完成。
-   批量上传混合链路：命中+未命中+失败项。
-   滚动窗内多进度条性能（例如 20 条）。

## 8.3 自动化回归

建议新增前端状态映射单测（纯函数）覆盖：

-   阶段到进度百分比映射
-   错误码到用户文案映射
-   批量汇总统计计算

## 10. 风险与缓解

1. **进度计算不精确**
    - 缓解：按阶段占比映射，明确“阶段进度”而非字节级精确值。
2. **批量条目太多导致卡顿**
    - 缓解：虚拟列表或节流刷新（MVP 可先做节流）。
3. **无宽高数据导致样式异常**
    - 缓解：未知类型回退样式 + 后续回填。

---

## 11. 验收标准（DoD）

1. 展示页能按图片比例自动应用样式分档。
2. 上传页默认不再显示调试文字红框。
3. 上传页验证码通过后不再显示“人机认证成功”提示文案。
4. 单图与批量上传均以进度条反馈主流程。
5. 批量上传多进度条位于固定高度滚动窗中，不挤占主视区。
6. 上传页昵称 placeholder 与 v6 文案约定一致，且不再显示冗余默认昵称提示。
7. 文件选择按钮风格与页面主色一致，视觉不突兀。
